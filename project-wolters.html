Scope

The challenge

Context

Two products, one root problem: content without structure.

Both the TAA Portal and One Web were built in Salesforce with multiple teams contributing content independently, no governance, no naming standards, no IA principles. The result: redundant navigation, empty pages, inconsistent terminology, and users unable to find what they needed.

My work across both products meant operating with full autonomy, planning the process, setting timelines, sourcing resources, and managing stakeholder relationships at a senior level, all without a design manager in the loop.

Impact
🗂️

Identified as implementation priority

TAA redesign was received with strong stakeholder approval and flagged as a priority for implementation, the clearest signal of impact in a project without live metrics.

🧭

Shifted stakeholder mindset

One Web evaluation reframed the conversation from content-filling to strategic UX, moving stakeholders from "we just need to organize content" to "we need to design with principles."

⚙️

Full ownership, senior-level access

Operated with complete autonomy across both products, planning process, setting timelines, managing senior stakeholders directly, with no design manager intermediary.

What I did
1

TAA, Information Architecture redesign

Mapped the full existing IA tree, identified structural issues, navigation depth, duplicate content, unclear naming, empty pages, and proposed a new architecture reducing redundancy and improving findability.

  • Mapped full existing IA tree, identified depth issues, duplicate content, unclear naming, empty pages
  • Proposed new architecture reducing redundancy and improving findability
  • Validated with cross-functional stakeholders across multiple rounds
  • Facilitated taxonomy workshop to define naming conventions, content, technical, and business stakeholders

IA mapping, current state diagnosis and proposed restructure across all TAA portal levels.

2

TAA, Layouts & component redesign

Redesigned the UX across all levels of the new IA, without creating new components. The constraint: reuse and adapt existing One Web components only, creating new variants where needed to meet TAA's specific requirements.

  • Mapped One Web components to TAA needs, identified which could be reused and where gaps existed
  • Designed new variants consistent with the parent system but adapted to TAA's different content context
  • Redesigned layouts across all IA hierarchy levels
  • Documented all use cases for developer handoff

Layout redesign, new page anatomy across hierarchy levels, using One Web component variants.

3

One Web, Expert UX evaluation

Led a structured heuristic evaluation of the One Web platform, WK's flagship product unifying services across countries. Designed the evaluation framework, trained evaluators, synthesized findings, and presented strategic recommendations to the extended team.

  • Platform scored 2.4/5 overall, lowest in Findability (1.8) and Control & Freedom (1.8)
  • Key issues: competing navigation elements, internal terminology exposed to users, inconsistent design standards
  • Designed evaluation framework, trained evaluators, synthesized findings across the team
  • Socialized findings to a proud team, reframed from "this is broken" to "here's the strategic path to excellence"
  • Presented redesign roadmap to the extended One Web team

Expert evaluation findings, heuristic scores by area and prioritized UX issues with design, content, and research recommendations.
