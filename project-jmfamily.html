The challenge

Context

A complex F&I platform serving dealers, buyers, and corporate users simultaneously.

JM Family's F&I Insights platform, branded as Gyde, supports Finance & Insurance managers across Toyota dealerships in the US. Every design decision had to balance three user types with competing needs, within a heavily regulated financial context and under significant organizational constraints.

My work spanned 5 workstreams across the product lifecycle: from research and expert evaluation to UX/UI delivery, Power BI data visualization, admin configuration flows, and cross-participation methodology.

What I did
1

PURE Analysis, Expert evaluation

Led a solo heuristic evaluation of the entire platform. Findings were presented to the design team, then to the client, then reviewed by the President of F&I Insights to define quarterly priorities.

  • Assigned by team lead, solo evaluator responsible for the full platform assessment
  • Interviewed designers from each track to understand technical constraints and business decisions behind existing UX choices
  • Mapped all issues, incorporated problems raised by designers, and scored each against PURE standards
  • Presented findings to internal design team, then to client stakeholders
  • Results reviewed by the President of F&I Insights, used to define Q priorities: quick fixes shipped immediately, strategic redesigns still in progress ~1 year later

PURE Analysis, scoring system, flow mapping, and findings presented to C-level stakeholders.

2

Deals Flows, UX/UI

Designed end-to-end flows for real-time deal configuration and buyer experience, covering success, error, and edge cases within an existing design system.

  • Supported requirements definition, raised usage scenarios, error states, edge cases with TL, BA, and PO
  • Created flow maps to confirm process and identify system query moments before UI design
  • Proposed UX reusing existing DS components, negotiated exceptions based on technical feasibility
  • Prototyped, documented interaction behaviors, and supported usability testing execution
  • Gathered critique feedback and iterated with design team before client presentation

Deal configuration interface with financial plan, products setup, and buyer cart.

3

Power BI Reports, UX/UI across two phases

Designed 9 report pages across Performance, Customer, and Process Adherence modules, navigating a blocked Phase 1 and a vindicated Phase 2.

  • Phase 1: PO without product vision blocked strategic design decisions, worked as executor without context, documenting questions and proposals throughout
  • Phase 2: New PO + UX lead took over, Phase 1 proposals adopted; testing with 5-7 users per report confirmed hypotheses and drove iteration
  • Key constraint: no custom DS components, worked within Power BI's native capabilities, guiding front-end devs to apply DS attributes (colors, radii, paddings) to PBI components
  • Defined IA of the Analytics section, collaborated with SD and PO on report goals, built prototypes simulating native PBI components
  • Participated as observer in usability testing; iterated based on findings

Power BI Reports, Customer dashboard with performance metrics, geographic distribution, and deal analytics.

4

Admin Settings, Self-service configuration

Designed the admin interface that allows dealer and corporate users to manage integrations, defaults, and product configurations, replacing dependency on external platforms.

  • Translated complex backend configurations (previously done by POs in external tools) into a self-service UI for admin users
  • Covered: activating/deactivating integrations, setting product defaults for sellers, dealer-level and corporate-level configurations
  • Challenge: making technical configuration tasks understandable and error-safe for non-technical admin users
  • Designed within existing DS, heavily table-based with contextual customization patterns

Admin settings, integrations management and self-service configuration interface.

5

UX Explorations, Translating ambiguity into design direction

Led iterative UX explorations across multiple product areas where requirements were undefined, using AI tools (Claude, Make, image generation) to accelerate ideation and turn ambiguity into structured design proposals for developers and product owners.

  • Worked across Recommendation Engine, Contracts, and Coverage module, each with limited specs and evolving requirements
  • Produced multiple iteration rounds per feature (7+ for Rec Engine alone), each surfacing gaps in product definition
  • Raised strategic questions about feature purpose, data dependencies, and user value before development began
  • Delivered structured, well-documented proposals that visual designers and devs could pick up and continue
  • Used Claude for ideation and content structuring, Make for workflow automation, and AI image generation tools (Banana/Firefly) for layout proposals and visual references

UX Explorations across Coverage, Recommendation Engine, and Contracts — iterative proposals using AI-assisted workflows.

6

Cross Participation

Built a framework to map design participation across all product areas, clarifying ownership, collaboration points, and design contribution across the full Lean + Agile process.

  • Mapped buyer journey across 5 stages: The Need, Introduction, Negotiation, Climax, Outcome
  • Defined product areas: Admin, Deals, Mirroring, Sign Contracts, Analytics & Reports
  • Clarified role of Buyer, Corporate, and Sales People at each stage
  • Documented design ownership (Owner vs Collaborator) across all 6 process phases
  • Used as internal reference and portfolio documentation for all design participation

Cross Participation, methodology map, buyer's journey, and product participation matrix across all platform areas.
